This project was less about inventing a complicated signal and more about building a reusable asset-management loop. I used factor regression to estimate portfolio inputs, optimized a long-only allocation, rebalanced every six months, and checked whether the resulting wealth path and turnover were reasonable. 重点和心得：这个项目不是为了堆复杂信号，而是把资产管理流程写成一个可以重复运行的框架。我用因子回归估计投资组合输入，再用 long-only MVO 优化权重，每半年再平衡，并检查财富路径和换手率是否合理。

Research Question 研究问题

The task was to design an algorithmic asset-management strategy that can be evaluated on different datasets. My version uses a factor-based return model and a constrained optimizer, then tests the strategy through a rolling portfolio simulation. 这个任务是设计一个可以在不同数据集上被评估的算法资产管理策略。我的版本使用因子模型估计收益和风险，再通过约束优化器生成权重，并用滚动回测检验组合表现。

Data

I tested the strategy on the third project dataset: 33 assets with monthly adjusted prices from December 1999 to December 2014, plus eight factor returns and a risk-free rate. The out-of-sample period begins after the first five years of history, and the portfolio is rebalanced every six months. 我使用第三组项目数据进行测试：33 个资产，月度复权价格覆盖 1999 年 12 月到 2014 年 12 月，并配套八个因子收益和无风险利率。样本外回测从前五年历史之后开始，组合每六个月再平衡一次。

Method 方法

The strategy first converts prices into excess returns, then estimates a factor model using the most recent 36 observations. The fitted model gives an expected return vector and covariance matrix for the optimizer. 策略先把价格转换为超额收益，然后用最近 36 个月的数据估计因子模型。模型输出预期收益向量和协方差矩阵，作为优化器的输入。

\[ R_t = \frac{P_t}{P_{t-1}} - 1,\qquad R^e_t = R_t - r_{f,t} \]
\[ R^e_{i,t} = \alpha_i + \beta_i^\top f_t + \epsilon_{i,t} \]

The portfolio step is long-only. I used the average estimated asset return as the target return, then minimized estimated variance under full-investment and non-negative-weight constraints. 投资组合优化是 long-only 的。我把平均估计收益作为目标收益，在满仓和非负权重约束下最小化组合方差。

\[ \min_x \frac{1}{2}x^\top Qx \quad \text{s.t.} \quad \mu^\top x \ge \bar{\mu},\quad \mathbf{1}^\top x = 1,\quad x \ge 0 \]

Turnover is tracked after each rebalance so the result is not judged only by the ending wealth. 每次再平衡后都会记录换手率，因此结果不只看最终财富，也看策略交易是否过于激进。

\[ \text{Turnover}_t = \sum_i \left|x_{i,t} - x^{\text{before}}_{i,t}\right| \]

Takeaway 我的理解

The result is a useful first version of an asset-management engine, but I would treat it as a controlled backtest rather than a finished trading system. The next step would be stricter robustness checks: transaction costs, sector concentration limits, alternative target-return rules, and comparisons against equal-weight and historical-mean baselines. 这个结果说明框架已经能跑通，但我会把它看成一个可控回测，而不是完整交易系统。下一步应该加入更严格的稳健性检验：交易成本、行业集中度限制、不同目标收益设定，以及和等权组合、历史均值模型进行对比。
